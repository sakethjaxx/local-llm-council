# ⚖️ Universal Council Engine

A futuristic, local web application that convenes a dynamic panel of AI agents to debate, cross-review, and deliver verdicts on any topic—from software architecture sprints to startup pitch decks.

![Council Engine UI Preview](council_preview.png) *(Note: Add a screenshot of the cyberpunk UI here!)*

---

## 🚀 Features

- **Universal Multi-Agent Reasoning**: Supply any context (code, text, proposals) and let multiple AI models analyze it from different perspectives.
- **Dynamic Seat Builder**: Add, remove, and configure council members entirely in the UI. Choose specific models (DeepSeek, Llama, Gemini, etc.), assign custom personas, and pick UI colors.
- **True Token Streaming & Markdown**: Watch the agents "type" out their analyses in real-time, instantly rendering beautiful Markdown inside a glassmorphic Cyberpunk UI.
- **Interactive Debate Mode**: Once the Chairman delivers a final verdict, drop into a chat window to interrogate specific council members using `@` mentions.
- **Markdown Export**: Download the entire generated session—analyses, peer reviews, verdict, and interactive debate—into a clean `.md` report for sharing.

---

## 🏗️ Architecture (3-Phase Asynchronous Pipeline)

We use **LiteLLM** to unify API calls to **OpenRouter**, allowing us to run top-tier models effectively for free (or extremely cheap) concurrently.

1. **Phase 1 — Independent Analysis**: Every dynamically defined agent analyzes the input topic in parallel based on their custom system persona.
2. **Phase 2 — Cross-Review**: Every agent reads the analyses generated by the others and provides a blunt critique.
3. **Phase 3 — Chairman's Verdict**: The designated Chairman reads all analyses and all peer reviews to deliver a final, synthesized decision.
4. **Phase 4 — Debate (Optional)**: Engage in multi-turn chat with any member of the council.

## ⚙️ Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure API keys

```bash
cp env.example .env
```
Edit `.env` and fill in your keys. By default, the system relies on **OpenRouter** to access multiple models easily.
```ini
OPENROUTER_API_KEY=sk-or-v1-...
```

### 3. Run the Server

```bash
python main.py
```

Open **http://localhost:8765** in your browser.

*(Note: The terminal running the server will print out the exact API interactions so you can see the prompts, retries, and responses flowing in real-time.)*

## 🛠️ Tech Stack

| Component | Tech |
|-----------|------|
| **Backend** | FastAPI + uvicorn (async) |
| **Streaming** | Server-Sent Events (SSE) with `asyncio.Queue` |
| **Routing** | LiteLLM (unifies OpenRouter, Groq, Ollama, OpenAI) |
| **Frontend** | Vanilla JS, CSS Grid, Glassmorphism, Marked.js |

## 🔧 Extending

Because the UI dynamically generates the council configuration, you no longer need to edit python code to change members! Just use the **Seat Builder** on the left panel to drop in new LiteLLM-compatible model strings (e.g., `groq/llama3-70b-8192` or `ollama/llama3`) and define new personas instantly.
